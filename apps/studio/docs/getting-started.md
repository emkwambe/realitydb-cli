# Getting Started with RealityDB Studio

RealityDB Studio is a domain-aware system simulation platform. Unlike traditional schema builders, it focuses on how data *behaves* over time, not just how it is stored.

## 1. The Workspace

The interface is divided into four main areas:
- **Left Sidebar**: Manage your project, templates, and global simulation settings.
- **Center Canvas**: Your visual modeling area where you draw tables and relationships.
- **Right Inspector**: Context-aware properties for tables, columns, and relationships.
- **Bottom Preview**: Real-time look at the synthetic data being generated by your model.

## 2. Your First Table

1. Click the **"New Table"** button in the top toolbar or sidebar.
2. Select the table on the canvas to open the **Inspector**.
3. Rename the table (e.g., `Users`).
4. Add columns using the **"Add Column"** button at the bottom of the table node.

## 3. Creating Relationships

Relationships in RealityDB are more than just foreign keys; they define system behavior.

1. Hover over a **Primary Key (Amber Key icon)** in a parent table.
2. Drag the connection handle to a child table.
3. In the popup, choose the **System Semantic** (e.g., "Trigger" or "Connection").
4. Decide if you want to automatically create a Foreign Key column.

## 4. Previewing Data

As you build, the **Preview Panel** at the bottom automatically generates "Ghost Rows"—realistic synthetic data based on your schema. This allows you to verify that your model produces the expected data distributions and relationships immediately.
